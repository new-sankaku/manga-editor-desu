# Manga Editor Desu! ロードマップ

プロジェクトの改良計画一覧。規模別に分類。

## 規模の定義
| 規模 | 目安 |
|------|------|
| S (小) | 単一ファイルの修正、バグ修正、即座に対応可能 |
| M (中) | 複数ファイルの修正、新機能追加、設計を伴う |
| L (大) | アーキテクチャレベルの変更、段階的移行が必要 |
| XL (特大) | プロジェクト全体に影響、長期的な取り組み |

## 推奨実施順序

1. **001** - ズームアウトバグ修正（1行修正）
2. **003** - デッドコード削除
3. **004** - Logger DRY化
4. **002** - Service Worker整理
5. **005** - コピペ複数対応

### 短期（機能改善）
6. **006** - レイヤーパネル差分更新
7. **007** - 自動保存
8. **008** - SVG遅延読み込み
9. **010** - エラーハンドリング統一
10. **012** - 画像エクスポート拡充

### 中期（リファクタリング）
11. **011** - Fabric.jsイベント整理
12. **009** - ショートカットカスタマイズ
13. **013** - Undo/Redoコマンドパターン化
14. **014** - index.html分割
15. **018** - アクセシビリティ
16. **019** - ComfyUIワークフロー改善

### 長期（アーキテクチャ）
17. **015** - ビルドパイプライン導入
18. **016** - ES Modules化（015が前提）
19. **017** - 自動テスト導入（015があると容易）
20. **020** - モバイル/タブレット対応
